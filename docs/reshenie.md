1. Алгоритма работы системы

1. Читается yaml-файл с детекторами.
    Для каждого детектора:
    1. КОмпилируется регулярное выражение
    2. Сохраняется структура Detector
2. Читается входная директория.
    Для каждого файла в директории:
    1. Создается горутина
    2. Открывается входной файл
    3. Создается выходной файл
3. Построчная обработка файла
    Для каждой строке в файле
    1. Проверка пустой строки - если да - пропуск
        1. Проверка включен ли детектор? - нет - пропуск
            1. Поиск всех совпадений в строке
                Для каждого совпадения:
                1. Определсяет, что заменять
                2. Формирует ключ для словаря замена
                3. Проверяет словарь замен
                    Ключ есть - используем существующую маску
                    Ключа нет - создаем новую маску и сохраняем в маппинг
                Заменяем в исходной строке исходное значение на маску
            Учитываем статистику
        1. Переход к следующему детектору
    1. Запись обработанной строки в выходной буфер.
   1. Перезод к следующей строке
   2. По окончанию файла - сброс буфера
4. Сбор статистики - вытаскивается из маппинга
5. Генерация отчета и его сохранение
